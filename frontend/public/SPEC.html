1. What it is

A multi-engine intelligence platform that turns raw accident signal (news, scanner audio, court filings, social posts) into qualified, contactable personal-injury leads.

  • Domain: accidentcommandcenter.com
  • Hosting: Vercel Pro
  • Database: Postgres + PostGIS + pgvector (via Vercel)
  • Email transport: Resend (alerts@caseflowplatform.com today; alerts@accidentcommandcenter.com pending DKIM)
  • SMS transport: Twilio (330-area-code number, A2P 10DLC brand approved)
  • Storage: S3-compatible blob (Vercel Blob)
  • Repo: github.com/ddigital14/accident-intel-platform

2. What it does (functional)

2.1 Ingest (cron, every 5–30 min)

  • Waze incident feed
  • TomTom traffic events
  • City 911 dispatch (Houston, Atlanta where available)
  • Scanner audio → Whisper / Deepgram transcription → AI summarization
  • News RSS aggregator (50+ regional outlets)
  • Reddit r/PoliceScanner, Citizen app
  • Police press releases (TX/GA/FL pages)
  • Hospital trauma boards
  • CourtListener / RECAP federal filings
  • State court searches (where indexable)
  • Funeral home / obituary feeds
  • GoFundMe accident campaigns
  • Tow company reports (where APIs exist)
  • TomTom DOI, county property assessors

2.2 Extract

  • Victim names (with deny-list filter — no journalists, officers, public figures)
  • Vehicle plates / VINs from press text
  • Injury severity, fatality count
  • Location → reverse-geocoded → city / state / county
  • Attorney mention detection (CourtListener cross-reference)
  • Insurance carrier names
  • Family relationships (obituary survivors parser)

2.3 Verify (Stage A regex + Stage B Claude)

  • Stage A: structural regex — rejects bylines, official titles, historical (WWII/Korean/Vietnam), out-of-US
  • Stage B: Claude Sonnet 4.6 verifier — yes/no/maybe, structured rationale
  • Persons enter victim_verified=TRUE only after both stages pass
  • Failure modes logged in victim_verifier_reason

2.4 Enrich (auto fan-out, on every persons UPDATE)

  • 30-engine matrix in lib/v1/system/auto-fan-out.js
  • DB trigger fires fan-out cascade on phone/email/address/name/dob/employer change
  • Per-engine 12s timeout, 55s total budget per pass
  • Rate limit: one full pass per person per 6h unless contact field changed
  • All field writes logged to enrichment_logs

2.5 Cross-check

  • Pairwise validation of phones (digits-only), emails (exact), addresses (70%+ token overlap), names (Levenshtein ≤2)
  • Conflicts dock confidence by 5–10 points
  • Matches log to evidence_cross_check_summary
  • States: single_source, no_evidence_yet, Nm/Mc (matches/conflicts), CONFLICT FLAGGED

2.6 Qualify

  • lead_score computed from evidence weight, recency, severity, contact completeness
  • Threshold: 120 = qualified, 80–119 = pending_named, <80 = pending
  • qualification_state on incidents
  • Auto-promotion when contact info added

2.7 Distribute

  • Slack alerts for high-value qualified leads
  • SMS alerts via Twilio (rep number, configured per metro)
  • Daily master-list email digest (Resend, multi-recipient)
  • Browser extension claim-lead button
  • Rep dashboard with one-click "Run All Sources" + 25 quick-link tools

5. What's enforced (rules)

  1. VICTIM-ONLY hard rule — name extractors must call applyDenyList(), must reject historical/byline/official names. Persons cannot become qualified without victim_verified=TRUE.
  2. Auto fan-out rule — every persons UPDATE auto-enqueues a cascade job that fires every relevant engine. DB trigger + cascade processor + ENGINE_MATRIX. Rate-limited 6h unless contact field changed.
  3. Enrichment_logs minimal schema — only person_id, field_name, old_value, new_value, created_at exist. Metadata folds into new_value JSON.
  4. Cross-engine consistency — every new engine MUST register in ENGINE_MATRIX, MUST log to enrichment_logs, MUST NOT bypass deny-list.
  5. Vercel Hobby constraints — 12 functions max, 11 crons max, no underscore prefix on api/ files, static require() only.
  6. Brand voice — confident, intelligent, slightly cinematic. Red (#EF4444) only for accident accent, DDS orange (#FF6600) only for parent-brand tagline.

10. Quick API surface

  • GET /api/v1/system/master-lead-list?secret=ingest-now&format=json&admin=true — current qualified leads
  • POST /api/v1/system/auto-fan-out?secret=ingest-now&action=run body {person_id} — fan out one
  • GET /api/v1/system/auto-fan-out?secret=ingest-now&action=batch&scope=qualified&limit=15 — fan out all
  • GET /api/v1/system/auto-fan-out?secret=ingest-now&action=matrix — engine matrix
  • GET /api/v1/enrich/evidence-cross-checker?secret=ingest-now&action=batch&force=qualified — cross-check
  • GET /api/v1/system/health — pipelines + cron + error counts
  • POST /api/v1/system/resend?action=send-html body {to:[],subject,html} — programmatic email
