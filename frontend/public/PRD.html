02 / EXECUTIVE SUMMARY

Executive Summary

ACC turns 22+ public data streams into qualified personal-injury leads with verified contact info — in minutes, not weeks.

The problem

  • Victims are unreachable. Police reports take 7–14 days. By then, a competing firm has already signed.
  • Manual research is slow. A paralegal averages 18 minutes per lead with a 22% hit rate.
  • Fake leads waste rep time. Existing platforms surface journalists, witnesses, and officials as leads.

The solution

A 60-engine AI-orchestrated pipeline with the VICTIM-ONLY data integrity rule — the only platform that mathematically refuses to enrich anyone except the actual victim. Opus 4.7 deep-phone-research orchestrator iterates four cycles, cross-validates across 17 enrichment sources, and delivers verified contact info as a clean queue.

04 / THE SOLUTION

How ACC Works

Five stages. Each hands off via Postgres triggers and an event-driven cascade queue.

1 · Ingest

22 active sources poll every 5–15 minutes: TomTom traffic, Waze Live Map, OpenMHz police scanners (Deepgram + Whisper), 25+ local-news RSS feeds, NewsAPI.org, 12 PD press release pages (NYPD, LAPD, Chicago, Houston, Atlanta, Dallas, Seattle, SFPD, Cincinnati, Boston, Miami, Philly), Citizen, GoFundMe, NextDoor, obituaries (Tributes.com, Legacy.com), funeral homes, hospital trauma boards, OSHA, FARS, NHTSA, CDC WONDER, FBI UCR, CourtListener, state DOT crash feeds (TX/GA/FL), open-data portals (Chicago + 4 others), Reddit, Telegram police channels.

2 · Verify (the integrity layer)

Stage A — regex deny-lists reject journalists, officers, spokespersons, witnesses, mayors. Stage B — Claude Sonnet 4.6 reads each candidate name with full article context and assigns role: victim · witness · journalist · official · family. Only victim rows persist with persons.role='victim' CHECK constraint.

3 · Enrich

17 engines fan out per verified victim: PDL Pro identify + enrich, Apollo people-match, voter rolls (FL/GA/TX/MO), homegrown OSINT miner (12 signals), Google CSE, Brave Search, Maricopa + Cook + Harris + LA + Fulton + Miami-Dade property records, Hunter, NumVerify, Trestle reverse phone + CNAM, Searchbug, Tracerfy, multi-language detector, plate-OCR vision, funeral-home survivors graph.

4 · Reason

Opus 4.7 deep-phone-research orchestrator runs a 4-cycle iterative reasoning loop. Reads all source_reports + enrichment_logs for a person, cross-validates phones against name + address + state, identifies missing fields, and dispatches engines likeliest to fill them. PDL identity-match gate prevents wrong-state Patrick Ramsey from being merged with right-state Patrick Ramsey.

5 · Deliver

  • Live Feed dashboard auto-refreshes every 30s
  • Daily Master Lead List email via Resend at 8:00 AM ET
  • Real-time SMS alerts via Twilio for score ≥ 85
  • Mobile-app push notifications
05 / DIFFERENTIATION

The Moat

1 · VICTIM-ONLY data integrity rule

The only platform that refuses to enrich journalists, officials, witnesses, or family. Encoded as a Postgres CHECK constraint — not a guideline. Every competing platform surfaces these as leads. We don't.

2 · Multi-language coverage (10 languages)

Spanish, Vietnamese, Mandarin, Korean, Tagalog, Haitian Creole, Russian, Arabic, Portuguese, French. Each language has its own seeded source list, native-speaker keyword variants, and a translation layer feeding back into the standard pipeline. Underserved communities equal uncontested deal-flow.

3 · Cross-Engine Intelligence telemetry

Every engine call bumps a counter in engine_capabilities. We can answer which engine produced the most qualified-leads-per-dollar this month in one query — and route resources accordingly. The system literally measures itself getting smarter.

4 · 17 enrichment sources orchestrated by Opus 4.7

Competitors hardcode 1–2 paid APIs. We orchestrate 17 with an AI that decides which to call next based on what's missing, then runs a 4-cycle reasoning loop. We fill contact info competitors call dead.

5 · VoyageAI semantic dedup

Voyage-3-large embeddings index every source_report. When two sources mention "Patrick Ramsey" in different states, embedding similarity + identity-match gate prevents collision. Competitors merge them and corrupt the lead.

6 · 60+ engines, 1 router, 2 Vercel functions

Vercel Hobby caps serverless functions at 12. We collapsed 49 endpoints into a single static-require() router. Total cost: $20/mo Vercel Pro. Equivalent on AWS Lambda + ALB: $400–$2,000/mo.

  • Export PDF / CSV / push-to-CRM.

7.7 Admin Tab (admin role only)

  • User management.
  • API key configuration via system_config (encrypted at rest).

10 / CROSS-ENGINE INTELLIGENCE

Cross-Engine Intelligence Architecture

The CEI loop

Every engine call triggers:

  1. engine_capabilities.invocation_count += 1
  2. On success: success_count += 1, success rate recomputed.
  3. Voyage embedding indexed into vector_cache for the source_report.
  4. cascade_queue enqueues downstream engines based on what new data was discovered.
  5. Auto-promote logic re-evaluates qualification_state.

13 / SECURITY & COMPLIANCE

Security & Compliance

  • TCPA litigator pre-check before any SMS — opt-out + DNC scrub.
  • VICTIM-ONLY rule — Postgres CHECK constraint on persons.role.
  • PDL identity-match gate — embedding similarity + name+state required to merge.
  • API keys in system_config (encrypted at rest), never committed.
  • Vercel function memory isolation — clean container per invocation.
  • Rate limiting — 30 req/min per IP on dashboard counts endpoints.
  • Audit log — enrichment_logs records every contact attribution.
  • GDPR — not currently EU-targeted (US-only).
  • Data retention — state-specific PI statute-of-limitations windows, then purged.
